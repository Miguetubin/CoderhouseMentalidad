# Cómo mejoré mi Proyecto Final de QA Manual hasta lograr su aprobación

Autor: Miguel Tubin

Curso: Mentalidad y Crecimiento

Diplomatura: Desarrollo Web Full Stack

Fecha: Junio 2026

## Contexto

Durante el cursado de la Diplomatura en Desarrollo Web Full Stack de Coderhouse, realicé el Proyecto Final de Testing QA Manual. El objetivo consistía en diseñar casos de prueba, documentar defectos, realizar pruebas de API utilizando Postman, ejecutar pruebas no funcionales mediante Lighthouse y elaborar un informe final integrador.

Al comenzar el proyecto contaba con una parte de los casos de prueba realizados, pero todavía faltaban varios componentes importantes para cumplir completamente con los requisitos de la entrega.

---

## Problema

Uno de los principales desafíos fue identificar qué elementos faltaban para completar el proyecto de forma profesional y coherente.

Durante la revisión inicial detecté que:

- Faltaban casos de prueba negativos.
- No se habían documentado reportes de defectos.
- No se habían realizado pruebas de API con Postman.
- No existía un informe Lighthouse.
- No se había elaborado un reporte final integrador.

Además, fue necesario mejorar algunos aspectos de consistencia y documentación para que todas las secciones del proyecto mantuvieran una misma estructura y trazabilidad.

---

## Acciones

Para resolver estos desafíos se realizó una revisión completa del proyecto y se definió un plan de trabajo dividido en etapas.

### Casos de prueba

Se completaron los casos negativos necesarios para complementar los escenarios positivos ya existentes. Esto permitió validar comportamientos esperados y situaciones de error.

### Reporte de defectos

Se documentaron distintos defectos simulados siguiendo buenas prácticas de QA, incluyendo:

- Severidad.
- Prioridad.
- Precondiciones.
- Pasos para reproducir.
- Resultado esperado.
- Resultado obtenido.

### Pruebas de API

Se realizaron pruebas utilizando Postman y la API pública Pokémon API para validar:

- Consultas exitosas.
- Manejo de errores.
- Respuestas HTTP.
- Estructura de datos.

### Pruebas no funcionales

Se ejecutó un análisis con Lighthouse para evaluar:

- Performance.
- Accessibility.
- Best Practices.
- SEO.

Los resultados obtenidos permitieron comprender la importancia de las pruebas no funcionales dentro del ciclo de calidad del software.

### Documentación

Finalmente se elaboró un reporte integrador para resumir el trabajo realizado y documentar los resultados obtenidos durante todo el proceso.

---

## Aprendizajes

Uno de los aprendizajes más importantes fue comprender que la calidad de un proyecto no depende únicamente de ejecutar pruebas, sino también de documentar correctamente cada actividad realizada.

También aprendí la importancia de:

- Mantener trazabilidad entre casos de prueba y defectos.
- Comunicar hallazgos de manera clara.
- Validar tanto aspectos funcionales como no funcionales.
- Organizar la información de forma estructurada.
- Revisar continuamente el trabajo para detectar oportunidades de mejora.

Además, comprendí que los errores detectados durante el proceso representan oportunidades de aprendizaje y mejora continua.

---

## Reflexión sobre Feedback Radicalmente Sincero

Durante el desarrollo del proyecto recibí observaciones relacionadas con la consistencia de la documentación y la necesidad de completar algunos requisitos faltantes.

Aplicar una mentalidad abierta al feedback me permitió identificar aspectos que inicialmente había pasado por alto y mejorar significativamente la calidad final del trabajo.

En lugar de interpretar las observaciones como críticas negativas, las utilicé como una herramienta para perfeccionar el proyecto. Este enfoque me ayudó a fortalecer mi capacidad de análisis, documentación y resolución de problemas.

Considero que uno de los mayores aprendizajes fue entender que el feedback sincero y constructivo es una de las herramientas más valiosas para crecer profesionalmente dentro de equipos tecnológicos.

---

## Conclusión

La experiencia de completar y mejorar el Proyecto Final de QA Manual me permitió desarrollar habilidades relacionadas con testing, documentación, análisis de problemas y comunicación técnica.

Más allá de los conocimientos técnicos adquiridos, el proyecto reforzó la importancia de la mejora continua, la capacidad de adaptación y la disposición para aprender a partir de los errores y del feedback recibido.

Estos aprendizajes resultan fundamentales para el desarrollo profesional dentro de la industria tecnológica y representan una base sólida para futuros desafíos.
